# 🐾 VRAMaぴょん 使い方マニュアル

[📖 Manual (English)](MANUAL_EN.md) ｜ **日本語**

---

## VRAMaぴょんってなに？

AI（ローカルLLM・画像生成）を使ってるとVRAMがどんどん埋まるけど、**GeForce + Windows では `nvidia-smi` がプロセス別のVRAMを `[N/A]` でしか返さない**（WDDMドライバの仕様）。だから「合計」は見えても「**誰が食ってるか**」が分からない。

VRAMaぴょんは、Windowsの性能カウンター `\GPU Process Memory(*)\Dedicated Usage` を読んで、**プロセス別のVRAM**と**どのGPUに乗ってるか**を見える化するツールだよ。さらに、Ollamaのモデル・画像生成のVRAM・プロセスそのものを「**そこだけ**」手放せる。

---

## 起動方法

```bash
python VRAMaPyon.py
```

- 初回は `customtkinter` が無ければ自動でインストールするよ
- GUIなしでデータだけ見たいときは `python VRAMaPyon.py --probe`（コンソール表示）

## 画面の見方

![VRAMaぴょんのメイン画面](完成_実機フル表示.png)

### GPUゲージ（上の2枚）

NVIDIAカードと内蔵GPU、それぞれの「全体メモリ使用率」を表示（タスクマネージャーの“GPUメモリ”相当）。

| 表示 | 意味 |
|---|---|
| 🟢 余裕 / 🟡 注意 / 🟠 警告 / 🔴 危険 | 使用率の色マーク |
| 専用 X/Y GB ・ 共有 X/Y GB | 専用VRAMと共有メモリ（RAM側）の内訳 |

> NVIDIAカードは合計を `nvidia-smi` から、内蔵GPU（AMD/Intel）は性能カウンター `\GPU Adapter Memory` から取得。内蔵は専用が満タンでも、あふれ先は共有（RAM）なのでNVIDIAには移らないよ。

### プロセス一覧（VRAMを使ってる順）

各行に、プロセス名・PID・VRAM使用量・操作ボタンが並ぶよ。

| 表示 | 意味 |
|---|---|
| 🤖 | AI関連アプリ（Forge / ComfyUI / Ollama / llama.cpp / KoboldCpp など） |
| 🖥5070 / 🍃内蔵 | そのプロセスが実際に乗ってるGPU（複数にまたがる時はGBで内訳） |
| 🔒 保護 | `dwm` / `explorer` / `csrss` などWindows重要プロセス（終了ボタンは出ない） |
| 自動 / 固定→◯◯ | 🎛GPU指定の今の状態（ボタンを押さなくても常時表示） |

「**AIだけ表示**」スイッチで、AI関連プロセスだけに絞り込めるよ。

### 🐾 足型スパークライン

各行の名前の後ろに、そのプロセスのVRAM推移を描く小さなグラフ（先端に🐾肉球）。一覧ヘッダー右のセグメントで2モードを切替：

| モード | 見え方 |
|---|---|
| 📊 全体比率 | GPU総量に対する占有率がそのまま高さ（張り付くと上端に貼り付く＋天井の点線） |
| 📈 細かい上下 | そのプロセス自身の最小〜最大で拡大（小さな増減もよく見える） |

線・肉球・数字の色は状態で変わるよ：**🔥張り付き（赤）/ ▲増加（オレンジ）/ 安定（ピンク）**。文字は白フチ付きで、グラフの上でも読めるようにしてある。

## 🦙 Ollamaのモデルを優しく解放

🦙パネルに、ロード中のOllamaモデルが一覧表示されるよ。

- **解放** … `keep_alive:0` を送って、そのモデルだけVRAMから降ろす。**Ollamaサーバーは生かしたまま**なので、次の会話ですぐ復帰する（再起動不要）
- **全部解放** … ロード中のモデルをまとめて降ろす
- 各モデルに **VRAM乗り率バッジ**（✅全部GPU / ⚠️RAM溢れ / 🔴ほぼRAM）が付くよ

Ollamaが起動していない時は「未接続」でボタンは灰色（繋がれば自動で有効化）。

---

## ❌ プロセスを終了する

AIアプリでもブラウザでも、各行の「終了」ボタンで終了できるよ（確認ダイアログ付き）。

- **🔒 保護機能**：`dwm` / `explorer` / `csrss` などWindows重要プロセスは、そもそも終了ボタンを出さないから安心
- 終了できなかった時は権限不足の可能性（管理者権限で実行してみてね）

---

## 🎨 画像生成（SD）のVRAMを解放

画像生成ツールのVRAMもワンクリックで解放できるよ。

| 対象 | 動作 | 必要な準備 |
|---|---|---|
| **Forge** | チェックポイントをアンロード | `--api` 付きで起動（既定 `http://127.0.0.1:7860`） |
| **ComfyUI** | モデル＆メモリを解放 | 起動していれば自動検知（既定 `http://127.0.0.1:8188`） |

未起動・未導入なら「繋がってないよ」でボタンは灰色。起動すれば自動で有効になるよ。

## 🎛 使うGPUを指定する（LLM以外を内蔵GPUへ）

各プロセスの行の **🎛GPU** ボタンから、そのアプリが使うGPUを指定できるよ。

| 選択肢 | 意味 |
|---|---|
| 🚀 RTX5070（高パフォーマンス） | NVIDIAカードで描画 |
| 🍃 内蔵GPU（省電力・VRAM節約） | 内蔵GPUへ逃がす |
| 🪟 自動 | Windowsまかせ |

これはWindowsの「グラフィック設定」と同じ（`HKCU\…\DirectX\UserGpuPreferences` を書く）。ブラウザやGUIアプリを内蔵GPUに逃がして、NVIDIAカードのVRAMをLLM/SDに空ける用途だよ。

**効き方の注意（大事）**
- 動くのは**描画系(D3D)のVRAMだけ**。**LLM/SDのCUDAメモリは常にNVIDIAカード**で、この指定では動かない
- 反映は**対象アプリの再起動後**・指定は**exeパス単位**（同じ `python.exe` の複数アプリは分けられない）
- 設定はPC再起動後も残る（Windowsのグラフィック設定そのもの）

> 💡 **本当の全振りはモニターのケーブルをマザボ側（内蔵GPU出力）に挿し替えること。** デスクトップ表示が `dwm` ごと内蔵GPUに移り、NVIDIAカードが計算専用になるよ。

## 🐾 フロート小窓

**🐾 小窓** ボタンで、枠なし・常時最前面の小さなウィジェットに変身（ライブのVRAMゲージ）。

- ドラッグで移動・**ダブルクリックで本体に復帰**・右クリックで「開く / 終了」メニュー
- 枠の色は使用率連動：<80%=ピンク / 80〜89%=アンバー / 90%〜=**赤くふわっと明滅**（おしゃれな警告メーター）

画面の隅に置いて、ながら見できるよ。

## 📸 記録（スナップショット）＋推移グラフ

**📸 記録** ボタンで、今の状態を Excel（`.xlsx`）に保存するよ。

- 1行＝1プロセスのフラットな表（取得時刻・GPU情報・プロセス名・VRAM・乗ってるGPU・モニター…）＝並べ替え／フィルタ／ピボットがそのまま効く
- 保存のたびに **「推移グラフ」シート** を作り直して、折れ線3本を**シート上部**に自動生成：
  - GPU全体VRAMの推移（使用 / 合計）
  - GPU使用率%の推移
  - プロセス別VRAMの推移（ピークの多い上位プロセス）
- データ表のプロセスVRAM列には**セル内データバー（ピンク）**＝大小がひと目で
- 既定ファイル名は**日付入り**（`VRAMaぴょん_記録_YYYYMMDD.xlsx`）。同じ日は1ファイルにたまる。既存ファイルを選べば追記
- 保存後に「**このファイルを今すぐ開く？**」と聞いてくれるよ

> 💡 グラフ（折れ線）は、1ファイルにスナップショットが**2点以上**たまって初めて線になるよ。1回だけだと点が1個で線が引けないので、何回か記録するか、次の自動記録を使ってね。

---

## ▶️ 自動記録

数分の推移をラクに撮りたいときはこれ。操作行の **▶️ 自動記録** ボタンと、その右の**間隔メニュー（15 / 30 / 60秒）**を使うよ。

1. **▶️ 自動記録** を押して、保存先を1回だけ選ぶ
2. あとは設定した間隔ごとに、同じファイルへ自動で追記（ボタンが赤い「⏹ 記録終了！(回数)」に変わる）
3. **⏹ 記録終了！** で停止
4. できたExcelの **「推移グラフ」タブ** を開けば、折れ線が育ってる📈

- データは内部のバッファに貯めて、実ファイルへの書き込み＆グラフ再構築は数回に1度＋停止時だけ＝**長時間回してもファイルで重くならない**
- 記録ファイルをExcelで開いたまま等で保存に失敗したら、自動で止まって教えてくれるよ

## ⚙️ 設定（`VRAMaPyon.py` 冒頭）

ファイル先頭の定数を書き換えれば調整できるよ。

| 定数 | 説明 | 既定 |
|---|---|---|
| `OLLAMA_URL` | Ollamaのアドレス | `http://localhost:11434` |
| `FORGE_URL` | Forge（`--api`起動が必要） | `http://127.0.0.1:7860` |
| `COMFY_URL` | ComfyUI | `http://127.0.0.1:8188` |
| `REFRESH_MS` | 自動更新の間隔（ミリ秒） | `2500` |

自動記録の間隔は、アプリ上の間隔メニュー（15 / 30 / 60秒）でいつでも変えられるよ。

## よくある質問

**Q. プロセス一覧が空っぽ**
→ 管理者権限で実行してみてね（一部のプロセスのカウンターは権限が要ることがあるよ）。

**Q. Ollama / 画像生成のボタンが灰色のまま**
→ まだ繋がってないだけ。Ollama / Forge（`--api`）/ ComfyUI を起動すれば、自動で有効になるよ。

**Q. グラフ（折れ線）が出ない・空っぽ**
→ ①Excelの下の **「推移グラフ」タブ** を見てる？（既定でそこが開くよ）②折れ線はスナップショットが**2点以上**で出るよ（1回だけだと点1個で線にならない）。

**Q. 自動記録がエラーで止まった**
→ 記録ファイルをExcelで開いてると保存に失敗するよ。閉じてからもう一度どうぞ。

**Q. 🎛GPU指定が効かない（LLMのVRAMが内蔵に移らない）**
→ 仕様です。動くのは描画系VRAMだけで、LLM/SDのCUDAメモリは常にNVIDIAカードに乗るよ。表示そのものを内蔵に移したいなら、モニターのケーブルをマザボ側へ。
